# Plan 01-01: Theme Foundation & Typography

## Objective

Establish the visual foundation for the landing page: update fonts (Space Grotesk + Inter), define color theme variables in Tailwind, and create the LandingLayout wrapper. This plan unblocks all subsequent UI work.

## Requirements

- LP-05: Bold, Energetic Visual Style

## Success Criteria

1. Root layout loads Space Grotesk (headings) and Inter (body) fonts via `next/font/google`
2. `globals.css` defines the full color palette as Tailwind theme variables via `@theme inline`
3. Existing Geist fonts are removed from root layout (replaced by Space Grotesk + Inter)
4. `LandingLayout` server component wraps landing page sections
5. `npm run typecheck` passes
6. `npm run lint` passes

## Implementation Steps

### Step 1: Update Root Layout — Replace Fonts

**File:** `src/app/layout.tsx`

- Remove `Geist` and `Geist_Mono` imports from `next/font/google`
- Add `Space_Grotesk` and `Inter` imports from `next/font/google`
- Create font instances:
  - `spaceGrotesk` with variable `--font-space-grotesk`, subsets `["latin"]`
  - `inter` with variable `--font-inter`, subsets `["latin"]`
- Update `<html>` className to use `${spaceGrotesk.variable} ${inter.variable}` instead of `${geistSans.variable} ${geistMono.variable}`
- Keep `h-full antialiased` classes
- Keep existing metadata (title, description)

### Step 2: Update Global CSS — Theme Variables

**File:** `src/app/globals.css`

- Update `:root` variables:
  - `--background: #ffffff`
  - `--foreground: #171717` (neutral-900)
- Update `@theme inline` block:
  - Map `--font-sans` to `var(--font-inter)` (body font)
  - Map `--font-heading` to `var(--font-space-grotesk)` (heading font)
  - Map color tokens:
    - `--color-background`: `var(--background)`
    - `--color-foreground`: `var(--foreground)`
    - `--color-hero-start`: `#1e3a8a` (deep blue)
    - `--color-hero-end`: `#7c3aed` (purple)
    - `--color-accent`: `#22c55e` (bright green)
    - `--color-text-primary`: `#171717`
    - `--color-text-secondary`: `#6b7280`
    - `--color-text-on-dark`: `#ffffff`
    - `--color-nav-solid`: `#ffffff`
    - `--color-destructive`: `#ef4444`
- Keep all existing print styles unchanged
- Update `body` font-family to reference the new `--font-inter` variable

### Step 3: Create LandingLayout Component

**File:** `src/components/landing/landing-layout.tsx`

- Create a server component (no `"use client"` directive)
- Wrap children in a `<div>` with appropriate layout classes
- Purpose: consistent wrapper for all landing page sections
- Export as default

```tsx
// ─────────────────────────────────────────────────────────────
// Landing Layout
// ─────────────────────────────────────────────────────────────
// Server component wrapper for landing page sections.
// Provides consistent layout structure.
// ─────────────────────────────────────────────────────────────

interface LandingLayoutProps {
  children: React.ReactNode;
}

export default function LandingLayout({ children }: LandingLayoutProps) {
  return <div className="min-h-screen">{children}</div>;
}
```

### Step 4: Verify

- Run `npm run typecheck` — must pass
- Run `npm run lint` — must pass
- Run `npm run build` — must pass
