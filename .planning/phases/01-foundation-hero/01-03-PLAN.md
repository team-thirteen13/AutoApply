# Plan 01-03: Hero Section with Gradient & CTA

## Objective

Build the hero section: full-width gradient background (deep blue → purple), centered text with headline/subheadline, CSS gradient blobs for depth, and a prominent sign-up CTA button. Includes fade-in animation that respects `prefers-reduced-motion`.

## Requirements

- LP-01: Hero Section with Sign-Up CTA
- LP-05: Bold, Energetic Visual Style

## Success Criteria

1. Hero section fills the viewport with a gradient background (`#1e3a8a` → `#7c3aed`)
2. Centered headline: "Build Resumes That Get You Hired" (Space Grotesk, Display size)
3. Centered subheadline below headline (Inter, Body size, lighter color)
4. Sign-up CTA button: "Sign Up for Free" (gradient variant, links to `/register`)
5. CSS gradient blob shapes visible behind text (low opacity, blurred)
6. Fade-in animation on page load (600ms opacity transition)
7. `prefers-reduced-motion: reduce` disables animation
8. Layout responsive: headline scales from 36px (mobile) to 48px (desktop)
9. `npm run typecheck` passes
10. `npm run lint` passes

## Implementation Steps

### Step 1: Create HeroBlob Component

**File:** `src/components/landing/hero-blob.tsx`

- Create a client component (`"use client"`) for CSS animation support
- Accept props: `className` (optional positioning/sizing overrides)
- Render a `<div>` with:
  - `position: absolute` (parent must be `relative`)
  - CSS `radial-gradient` background using hero colors
  - `filter: blur(80px)`
  - `opacity: 0.15`
  - Rounded shape via `border-radius: 50%`
- Export as named export

```tsx
"use client";

interface HeroBlobProps {
  className?: string;
}

export function HeroBlob({ className = "" }: HeroBlobProps) {
  return (
    <div
      className={`absolute rounded-full ${className}`}
      style={{
        background: "radial-gradient(circle, #7c3aed 0%, #1e3a8a 100%)",
        filter: "blur(80px)",
        opacity: 0.15,
      }}
    />
  );
}
```

### Step 2: Create Hero Component

**File:** `src/components/landing/hero.tsx`

- Create a client component (`"use client"`) for fade-in animation
- Use `useState` + `useEffect` for mount-based animation trigger:
  - On mount, set `isVisible = true` after a tick
- Use `useMediaQuery` or CSS `@media (prefers-reduced-motion: reduce)` to disable animation
- Render:
  - `<section>` with:
    - Full-width gradient background: `bg-gradient-to-br from-hero-start to-hero-end`
    - Full viewport height: `min-h-screen`
    - `relative overflow-hidden` (for blob positioning)
    - `flex items-center justify-center` (center content)
  - Inside: `<div>` with `max-w-6xl mx-auto px-4 text-center relative z-10`
    - Headline: `<h1>` with Space Grotesk font, Display size, white text
    - Subheadline: `<p>` with Inter font, Body size, white/80 opacity
    - CTA: `<Button>` with `variant="gradient"` and `size="lg"`, wrapped in `<Link href="/register">`
      - Text: "Sign Up for Free"
  - 3x `<HeroBlob>` positioned absolutely:
    - Top-left: large blob (w-96 h-96, top-20 -left-20)
    - Bottom-right: medium blob (w-72 h-72, bottom-20 -right-10)
    - Center: small blob (w-48 h-48, top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2)
- Animation: apply `transition-opacity duration-600` with opacity controlled by `isVisible` state
- Reduced motion: use CSS `@media (prefers-reduced-motion: reduce)` to set `animation: none` and full opacity

## QA Scenarios

### Scenario 1: Hero renders correctly
- **Steps:** Navigate to `/`
- **Expected:** Full-viewport gradient hero with centered headline, subheadline, and CTA button
- **Edge cases:** Slow network (fonts loading), no JS (server render fallback)

### Scenario 2: CTA navigates to sign-up
- **Steps:** Click "Sign Up for Free" button
- **Expected:** Navigates to `/register`
- **Edge cases:** Already logged in (should still go to register or dashboard)

### Scenario 3: Responsive layout
- **Steps:** View at 375px, 768px, 1024px widths
- **Expected:** Headline scales properly, no horizontal scroll, text readable
- **Edge cases:** Ultra-wide screens (1440px+), very small screens (320px)

### Scenario 4: Fade-in animation
- **Steps:** Load page, observe hero
- **Expected:** Content fades in over 600ms
- **Edge cases:** `prefers-reduced-motion: reduce` enabled — no animation, instant render

### Scenario 5: Blob shapes visible
- **Steps:** Look at hero background
- **Expected:** Subtle gradient blobs visible behind text, adding depth
- **Edge cases:** High contrast mode, color blindness

### Scenario 6: Build succeeds
- **Steps:** Run `npm run build`
- **Expected:** No type errors, no lint errors, production build completes
