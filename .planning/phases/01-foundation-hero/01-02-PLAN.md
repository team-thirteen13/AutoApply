# Plan 01-02: Sticky Navigation with Auth-Aware CTAs

## Objective

Build a sticky navigation bar that adapts based on authentication state: shows Sign Up / Sign In when logged out, Dashboard link when logged in. Transitions from transparent (over hero) to solid white on scroll. Includes a mobile hamburger menu with slide-from-right animation.

## Requirements

- LP-07: Navigation with Sign-Up / Sign-In Links
- LP-05: Bold, Energetic Visual Style

## Success Criteria

1. Navigation bar is sticky at the top of the viewport
2. Shows "AutoApply" logo text on the left
3. Auth-aware: logged out shows Sign Up + Sign In; logged in shows Dashboard
4. Transparent background over hero, transitions to solid white on scroll (10px threshold)
5. Mobile: hamburger icon opens slide-from-right menu overlay
6. Mobile menu has semi-transparent backdrop and closes on link click or backdrop click
7. `npm run typecheck` passes
8. `npm run lint` passes

- Create a client component (`"use client"`)
- Accept `user` prop (AuthUser | null) â€” passed from parent server component

  - Logged out: Sign In (ghost style), Sign Up (gradient button using Button component)

### Step 2: Create MobileNav Component

**File:** `src/components/landing/mobile-nav.tsx`

- Create a client component (`"use client"`)
- Accept props: `isOpen`, `onClose`, `user` (AuthUser | null)
- Render:
  - Overlay backdrop: fixed, semi-transparent black (`bg-black/50`), full viewport
  - Menu panel: fixed, right-0, top-0, full height, white background, w-80 max
  - Transition: `translate-x-full` â†’ `translate-x-0` when open (300ms ease-in-out)
  - Close button (lucide `X` icon) at top right
  - Nav links (same auth-aware logic as desktop)

- Backdrop click also triggers `onClose`

### Step 3: Integrate Navbar into Home Page

## QA Scenarios

### Scenario 1: Logged-out navigation
- **Steps:** Open page in incognito (not logged in)
- **Expected:** Nav shows "AutoApply" logo, "Sign In" link, "Sign Up" button (green/gradient)
- **Edge cases:** Nav renders correctly before JS hydration

### Scenario 2: Logged-in navigation
- **Steps:** Log in, navigate to `/`
- **Expected:** Nav shows "AutoApply" logo, "Dashboard" button
- **Edge cases:** Session expiry while on page

### Scenario 3: Scroll transition
- **Steps:** Load page, scroll down past 10px
- **Expected:** Nav background transitions from transparent to solid white, text from white to dark
- **Edge cases:** Rapid scroll, scroll to bottom then back to top

### Scenario 4: Mobile menu
- **Steps:** Resize to 375px width, tap hamburger icon
- **Expected:** Menu slides in from right with backdrop, shows auth links
- **Edge cases:** Open menu, scroll body (menu should prevent body scroll)

### Scenario 5: Mobile menu close
- **Steps:** Open mobile menu, tap a link or backdrop
- **Expected:** Menu slides out, page navigates (if link) or stays (if backdrop)
